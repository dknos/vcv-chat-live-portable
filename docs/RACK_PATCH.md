# VCV Rack patch and OSC mapping

## Validated patch

The canonical artifact is `patches/ChatRack-Live.vcv`. It is generated by `scripts/build-rack-patch.mjs`, not maintained by manually editing cables.

An additional loadable profile is kept separate so the canonical live rack is
never overwritten:

- `patches/Doom-Jazz-Machine.vcv` is generated by
  `scripts/build-doom-jazz-patch.mjs`. It keeps all 27 OSC channels and the
  existing piano, guitar, Scando solo, mute gate, heartbeat watchdog, and final
  limiter, then adds a bass-pitched four-oscillator Elemental Revelator,
  kick-sidechained Pressed Duck dynamics, Moon Phase Distortion, and Liminal Vast
  reverb. The added voice is multiplied by `/synth`, so the normal synth mix
  command controls the whole machine and `!mix synth 0` silences it. The result
  validates at 59 modules and 120 cables.

The Doom profile uses only modules installed on the target workstation. It is an
architectural translation of the reference rack; a literal copy would require
several unavailable Instruo, DrumKit, Autinn, nullpath, and wildergarden modules.

On the configured Windows workstation, switch profiles while OBS is stopped:

```powershell
scripts\load-rack-profile.ps1 doom
scripts\load-rack-profile.ps1 live
```

The loader syncs the chosen repository artifact to Rack's Windows patch folder,
requests a normal Rack close, and verifies the clean window title plus the exact
module/cable count. It never force-stops Rack and never starts or stops OBS. By
default it refuses to switch while OBS is running; `-AllowObsRunning` is an
explicit operator acknowledgement that recording and streaming are already
stopped. The Rack-only capture helper explicitly allowlists all three names.

The current build contains:

- Rack/Core 2.6.6, Fundamental 2.6.4, trowaSoft 2.0.9, Coalescent 2.2.1, CVfunk 2.0.47, and forsitan 2.7.0
- 54 modules and 98 cables
- `cvOSCcv` plus two 16-channel output expanders for 27 receive channels
- generated Scando physical-model lead, bass, kick, snare, and hi-hat voices
- macro smoothing, brightness/chaos modulation, a clock-synchronized stereo delay, and Operon-modulated Haze stereo texture
- bounded output clipping, a fail-closed master VCA, and a clock heartbeat watchdog
- configurable Windows audio output over WASAPI at 48 kHz/256 samples

Open the copied patch in VCV Rack Free 2.6.6. The Audio 2 module should report
the operator-selected device, 48 kHz, and 256 samples.
Its managed output lift is 1.60, applied after the 4.8 V safety clipper so the
stream remains audible on phones and small speakers without raising Windows'
global output volume.

For managed starts or restarts, use `scripts/manage-windows-rack.ps1`. It launches
Rack from its installation directory and refuses to report success until the
window title names `ChatRack-Live.vcv` and the live autosave contains exactly 54
modules and 98 cables. This prevents an ignored patch argument from silently
leaving a blank rack on stream.

## Rebuild the patch

From WSL:

```bash
npm run build:patch
npm run build:doom
```

The builder validates unique module IDs, unique cable destinations, reciprocal OSC-expander adjacency, the installed third-party module set, exact patch size, and all 27 OSC channels before writing the archive. It uses the Rack Free template at `C:\Program Files\VCV\Rack2Free\template.vcv` by default.

To target a renamed Windows audio endpoint, set `VCV_AUDIO_DEVICE` for the build or select the endpoint manually in Rack after opening the patch:

```bash
VCV_AUDIO_DEVICE='your Windows audio device' npm run build:patch
```

After rebuilding, copy the canonical artifact to the Windows Rack patch location before the next show.

## Embedded OSC configuration

The patch already embeds these trowaSoft settings:

- IP: `127.0.0.1`
- transmit port: `7000` (unused by this system)
- receive port: `7001`
- namespace: `/chat`
- reconnect on patch load: enabled

The module prepends its namespace, so its `/tempo` channel receives `/chat/tempo`.

## Channel contract

The bridge always uses the same 27-channel contract. Some state channels are consumed by the bridge's sequencer and exposed to Rack for status/future expansion; the audible patch directly cables the channels noted below.

| Ch. | OSC address | Current behavior |
|---:|---|---|
| 1 | `/chat/tempo` | Bridge sets sequencer/bar timing; present in Rack contract |
| 2 | `/chat/root` | Bridge converts root into generated pitch CV; present in Rack contract |
| 3 | `/chat/scale` | Bridge quantizes generated notes; present in Rack contract |
| 4 | `/chat/energy` | Directly drives synth/drum activity macros |
| 5 | `/chat/density` | Bridge controls event probability; present in Rack contract |
| 6 | `/chat/brightness` | Directly drives the synth filter macro |
| 7 | `/chat/space` | Directly drives delay send/blend |
| 8 | `/chat/chaos` | Drives pattern mutation and bounded Rack modulation |
| 9 | `/chat/scene` | Bridge scene state; present in Rack contract |
| 10 | `/chat/seed` | Deterministic generator seed; present in Rack contract |
| 11 | `/chat/drums` | Continuously controls the complete drum-layer mix, 0-10 V |
| 12 | `/chat/mutate` | Mutation event notification |
| 13 | `/chat/drop` | Directly ORed into the kick impact trigger |
| 14 | `/chat/mute` | Direct final master VCA: 10 V live, 0 V muted |
| 15 | `/chat/freeze` | Audience-control state notification |
| 16 | `/chat/change` | Fires after a committed state change |
| 17 | `/chat/panic` | Emergency event; bridge also sends mute=0 V and freezes controls |
| 18 | `/chat/note` | Lead pitch on VAL and lead gate on TRG |
| 19 | `/chat/bass` | Bass pitch on VAL and bass gate on TRG |
| 20 | `/chat/kick` | Kick trigger |
| 21 | `/chat/snare` | Snare trigger |
| 22 | `/chat/hat` | Hi-hat trigger |
| 23 | `/chat/accent` | Bounded lead accent CV |
| 24 | `/chat/clock` | Sixteenth-note clock for delay sync, random modulation, and watchdog |
| 25 | `/chat/piano` | Continuously controls sampled piano mix, 0-10 V |
| 26 | `/chat/guitar` | Continuously controls sampled electric-guitar mix, 0-10 V |
| 27 | `/chat/synth` | Continuously controls the generated synth mix, 0-10 V |

Performance macros and instrument mixes are 0-10 V, generated pitch is clamped to -4 to +4 V/oct, accent is 2-10 V, and trigger events use a value of 1. Pitch travels through the trowaSoft value outputs while gates and events use trigger outputs.

## Audio and fail-safe behavior

The bridge generates the exact scale-aware pitch and rhythm events, while Rack remains the fixed sound engine. The current patch uses a note-excited Scando physical-model lead, square bass, synthetic kick, filtered-noise snare, noise hi-hat, shared filter, clocked stereo delay, and a conservative Haze stereo texture. Operon's three slow phases modulate only Haze; they never affect chat-controlled timing, pitch, or gates.

Two independent controls must be healthy for output:

1. `/chat/mute` must be 10 V. The initial patch is silent until the bridge explicitly announces an unmuted state.
2. `/chat/clock` must continue. A one-second release envelope drives a second VCA, so a stopped bridge fades the output near silent in roughly one second.

`!rack panic` sets mute to 0 V, freezes audience control, clears the pending bar queue, and sends the panic trigger. Keep Rack's master/audio controls manually reachable as an additional operator stop.

## Acceptance test

1. Open the patch and confirm cvOSCcv reconnects on UDP 7001.
2. Start the Windows bridge in immediate mode for testing.
3. Exercise representative extremes through stdin or a test chat:

   ```text
   !energy 0
   !energy 100
   !brightness 0
   !brightness 100
   !space 100
   !chaos 100
   !mutate
   !drop
   ```

4. Confirm the configured audio output meters move without clipping.
5. Stop the bridge and confirm the heartbeat watchdog silences the patch.
6. Restart the bridge and confirm `/chat/mute` reopens the final VCA only after state synchronization.
